O que há de novo na v2

⠋
Feedback Visual em Tempo Real
Spinners animados para operações longas, barras de progresso por grupo de pacotes e timestamps ao final de cada módulo.
🪐
JupyterLab 4.x — Reescrito
Extensões compatíveis com Lab 4.x (jupyterlab-lsp≥5.0), instalação em batches, smoke tests automáticos ao final: version, kernelspec e import do pylsp.
📌
Conflitos de Dependências Resolvidos
Pins explícitos para rich<15.0 (prefect 3.x) e pathspec<0.13 (dbt-core 1.x). Grupos instalados na ordem correta.
🔍
Verificações Inteligentes
Antes de reinstalar qualquer ferramenta, o script verifica se já está presente. Espaço em disco e internet são checados antes de iniciar.
📋
Sumário Final de Falhas
Ao término, exibe três seções: versões instaladas, itens ignorados (já existentes) e falhas não-críticas com referência ao log.
🟩
Correção nvm + set -u
O bloco nvm desabilita set -u durante todo o escopo, corrigindo o erro "unbound variable" com arrays no Bash 5.0 (Ubuntu 20.04+).

JupyterLab 4.x — Smoke Tests

Ao final do módulo 10, o script executa três verificações automáticas para confirmar que o ambiente está funcional:

✅ Versão instalada

✅ Kernel Python registrado
jupyter kernelspec list
✅ Backend LSP importável
python3 -c "import pylsp"
🚀 Iniciar JupyterLab

Estrutura de Pastas

~/data-projects/ ├── notebooks/ # análises exploratórias (.ipynb) ├── data/ │ ├── raw/ # dados brutos — não commitar! │ ├── processed/ # dados tratados e limpos │ └── external/ # fontes externas ├── scripts/ # ETL e automações (.py) ├── reports/ # outputs finais (PDF, HTML, imagens) ├── dashboards/ # apps Streamlit / Dash ├── models/ # artefatos ML treinados (.pkl, .pt) └── template_projeto.ipynb # notebook base
